About World Time Zones

A comprehensive web application built with Go that provides real-time information about different countries, their time zones, and development statistics.

Core Features

Search System

Search countries by name, capital, or region with real-time results.

Filtering System

Filter countries by timezone names, regions (continents), and current times in the countries.

Pagination

Browse through country listings with 12 items per page for better navigation.

Favorites

Save and manage your favorite countries with persistent storage.

Data Sources

REST Countries API

Primary source for country data, providing:

name, capital, region, flag, timezones, population, area, languages, currencies, idd, car, borders

HDI Statistics

Human Development Index data from UNDP Human Development Report 2023/2024

Source: HDI Data Center

Leaflet.js API

Interactive mapping library used for geographical visualization

Source: Leaflet.js

Map Features

The interactive map is built using Leaflet.js and features two custom overlay layers:

Timezone Boundaries Overlay

Interactive timezone boundaries using data from the timezone-boundary-builder project (specifically the 'timezones-with-oceans-now.geojson.zip' file). Source: timezone-boundary-builder releases

Live Terminator Line Overlay

Custom-built overlay showing real-time day/night boundaries across the globe with an animated terminator line.

Technical Implementation

The timezone boundary data is processed using geojsplit to optimize performance. This splits the large GeoJSON file (timezones-with-oceans-now.geojson.zip) into smaller, more manageable chunks while maintaining full functionality.

Documentation

Initial Development

Started with the REST Countries v3.1 API integration, followed by creating multiple pages using vanilla HTML. Enhanced the UI with hover animations and theme implementations through CSS.

Map Integration

Integrated Leaflet.js with OpenStreetMap API for the interactive map. Implemented two key overlays: timezone boundaries (using timezone-boundary-builder data) and a custom-built live terminator line showing real-time day/night separation.

Development Process

Documented the journey through screenshots of key challenges, maintained comprehensive inline code comments, and tracked all external resources including APIs and statistical data sources.

Refinement

Gathered user feedback, conducted thorough testing, and fixed identified bugs. Concluded by restructuring the codebase into separate files for improved readability and optimization.
